# OpenClaw → Claude Code Migration Skill

This skill migrates an existing OpenClaw instance to run natively on Claude Code CLI.

## What gets migrated

1. **Project context .md files** — AGENTS.md, IDENTITY.md, SOUL.md, USER.md, MEMORY.md, TOOLS.md → merged into a system prompt file
2. **Heartbeat** — OpenClaw's built-in heartbeat → crontab + tmux send-keys
3. **Cron jobs** — OpenClaw's `cron/jobs.json` → macOS crontab entries + helper scripts
4. **Telegram bot token** — extracted from `openclaw.json` → environment variable for Claude Code's `--channels` flag
5. **Skills** — unchanged (SKILL.md format is the same), but validates frontmatter syntax

## What gets added (new)

1. **tmux session management** — attach/detach to the running Claude Code session
2. **Monitor script** — check if Claude Code is running, show status, memory usage

## Pitfalls & limitations

1. **Telegram slash commands not yet supported** — commands like `/status`, `/restart`, `/clear` sent from Telegram cannot be forwarded to Claude Code CLI yet
2. **Skill description must use double quotes** — `description: "text with: colons"` — unquoted colons break YAML frontmatter and the skill won't load
3. **Use crontab, not LaunchAgent, for cron jobs** — LaunchAgent is only for the main startup; individual cron jobs run via crontab for efficiency

---

## How to use

Paste this into your Claude Code session:

```
Read the SKILL.md file from this project and run the migration.
```

Claude Code will then:
1. Detect your OpenClaw installation
2. Ask you a few questions about your setup
3. Generate all the scripts and config files
4. Set up crontab entries
5. Test the setup

## Migration Steps

When this skill is triggered, follow these steps in order:

### Step 0: Detect OpenClaw Installation

Find the user's OpenClaw setup:

```bash
# Common locations to check
ls -la ~/.openclaw/openclaw.json 2>/dev/null
ls -la .openclaw/openclaw.json 2>/dev/null
```

If not found, ask the user:
> "I can't find your OpenClaw config. Where is your OpenClaw instance directory? (e.g., ~/my-assistant)"

Store the result as `OPENCLAW_DIR`.

Identify key paths:
- `WORKSPACE` = the directory containing AGENTS.md, IDENTITY.md, etc. (usually `$OPENCLAW_DIR/data` or `$OPENCLAW_DIR/workspace`)
- `CONFIG` = path to `openclaw.json`
- `STATE_DIR` = `.openclaw/` directory

### Step 1: Migrate Context .md Files → System Prompt

Read the existing .md files and create a system prompt generator script.

Create `$OPENCLAW_DIR/scripts/generate-system-prompt.sh`:

```bash
#!/bin/bash
# Auto-generated by openclaw-migration skill
# Merges OpenClaw identity .md files into a Claude Code system prompt

BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
WORKSPACE="$BASE_DIR/data"
OUTPUT="/tmp/$(basename "$BASE_DIR")-system-prompt.md"

cat > "$OUTPUT" << 'HEADER'
# Assistant — Claude Code Mode

You are running inside Claude Code CLI.
Follow the instructions in the identity files below.
HEADER

for mdfile in AGENTS.md IDENTITY.md SOUL.md USER.md MEMORY.md TOOLS.md; do
    if [ -f "$WORKSPACE/$mdfile" ]; then
        echo -e "\n---\n## ${mdfile%.md}" >> "$OUTPUT"
        cat "$WORKSPACE/$mdfile" >> "$OUTPUT"
    fi
done

echo "✅ System prompt: $OUTPUT ($(wc -c < "$OUTPUT" | tr -d ' ') bytes)"
```

Make it executable: `chmod +x scripts/generate-system-prompt.sh`

### Step 2: Extract Telegram Bot Token

Read the bot token from `openclaw.json`:

```python
import json

with open("OPENCLAW_CONFIG_PATH") as f:
    cfg = json.load(f)

# Try common locations in the config
tg = cfg.get("channels", {}).get("telegram", {})
token = (
    tg.get("botToken")
    or tg.get("accounts", {}).get("default", {}).get("botToken")
    or ""
)

if token:
    print(f"Found Telegram bot token: {token[:10]}...{token[-5:]}")
else:
    print("No Telegram bot token found in openclaw.json")
```

If found, save it to `$OPENCLAW_DIR/env.sh`:

```bash
export TELEGRAM_BOT_TOKEN="the-token-here"
```

If not found, ask the user if they want to set up Telegram.

### Step 3: Create Startup Script

Create `$OPENCLAW_DIR/scripts/claude-start.sh`:

```bash
#!/bin/bash
# Launch Claude Code with full OpenClaw feature set
# Auto-generated by openclaw-migration skill

BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
WORKSPACE="$BASE_DIR/data"
SESSION_NAME="$(basename "$BASE_DIR")"
SYSTEM_PROMPT="/tmp/${SESSION_NAME}-system-prompt.md"

# Source environment
[ -f "$BASE_DIR/env.sh" ] && source "$BASE_DIR/env.sh"

# Generate system prompt
"$BASE_DIR/scripts/generate-system-prompt.sh"

cd "$WORKSPACE"

# Build claude command
CLAUDE_ARGS=(
    --append-system-prompt-file "$SYSTEM_PROMPT"
    --dangerously-skip-permissions
)

# Add Telegram channel if token exists
if [ -n "$TELEGRAM_BOT_TOKEN" ]; then
    CLAUDE_ARGS+=(--channels plugin:telegram@claude-plugins-official)
fi

exec claude "${CLAUDE_ARGS[@]}"
```

### Step 4: Create tmux Wrapper (start.sh)

Create `$OPENCLAW_DIR/scripts/start.sh`:

```bash
#!/bin/bash
# Start Claude Code in a tmux session (for LaunchAgent or manual use)
# Auto-generated by openclaw-migration skill

BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
SESSION_NAME="$(basename "$BASE_DIR")"

# Prevent nested session errors
unset CLAUDECODE

# Source environment
[ -f "$BASE_DIR/env.sh" ] && source "$BASE_DIR/env.sh"

# Kill stale session
tmux kill-session -t "$SESSION_NAME" 2>/dev/null

# Start Claude Code in tmux
tmux new-session -d -s "$SESSION_NAME" "$BASE_DIR/scripts/claude-start.sh"

# Track activity for cron idle detection
tmux pipe-pane -t "$SESSION_NAME" -o \
    "while read -r _; do touch /tmp/${SESSION_NAME}-active; done"

echo "✅ Claude Code started in tmux session '$SESSION_NAME'"
echo "   Attach: tmux attach -t $SESSION_NAME"

# Keep process alive (for LaunchAgent)
while tmux has-session -t "$SESSION_NAME" 2>/dev/null; do
    sleep 5
done
```

### Step 5: Create Cron Helper Scripts

Create `$OPENCLAW_DIR/scripts/cronjob_tmux.sh`:

```bash
#!/bin/bash
# Send a cron task to the main Claude Code session (waits for idle)
# Auto-generated by openclaw-migration skill
# Usage: cronjob_tmux.sh <md_file> [idle_seconds] [max_retries]

MD_FILE="${1:?Usage: cronjob_tmux.sh <md_file> [idle_seconds] [max_retries]}"
IDLE_THRESHOLD="${2:-300}"
MAX_RETRIES="${3:-10}"

BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
SESSION_NAME="$(basename "$BASE_DIR")"
ACTIVITY_FILE="/tmp/${SESSION_NAME}-active"
MD_BASENAME="$(basename "$MD_FILE")"

if ! tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
    echo "$(date): Session not running, skipping"
    exit 0
fi

if [[ ! -f "$ACTIVITY_FILE" ]]; then
    tmux send-keys -t "$SESSION_NAME" "Read ${MD_BASENAME} and execute" Enter
    exit 0
fi

for ((attempt=1; attempt<=MAX_RETRIES; attempt++)); do
    LAST_MOD=$(stat -f %m "$ACTIVITY_FILE" 2>/dev/null || echo 0)
    NOW=$(date +%s)
    IDLE_TIME=$((NOW - LAST_MOD))

    if [[ $IDLE_TIME -ge $IDLE_THRESHOLD ]]; then
        echo "$(date): Idle ${IDLE_TIME}s → sending: ${MD_BASENAME}"
        tmux send-keys -t "$SESSION_NAME" "Read ${MD_BASENAME} and execute" Enter
        exit 0
    fi

    echo "$(date): Busy (idle ${IDLE_TIME}s), retry $attempt/$MAX_RETRIES..."
    sleep "$IDLE_THRESHOLD"
done

echo "$(date): Still busy after $MAX_RETRIES retries, giving up"
```

Create `$OPENCLAW_DIR/scripts/cronjob_isolated.sh`:

```bash
#!/bin/bash
# Run a one-shot Claude session for a cron task
# Auto-generated by openclaw-migration skill
# Usage: cronjob_isolated.sh <md_file>

MD_FILE="${1:?Usage: cronjob_isolated.sh <md_file>}"
BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
WORKSPACE="$BASE_DIR/data"
SESSION_NAME="$(basename "$BASE_DIR")"
SYSTEM_PROMPT="/tmp/${SESSION_NAME}-system-prompt.md"

[[ "$MD_FILE" != /* ]] && MD_FILE="$WORKSPACE/$MD_FILE"

[ -f "$BASE_DIR/env.sh" ] && source "$BASE_DIR/env.sh"
"$BASE_DIR/scripts/generate-system-prompt.sh" >/dev/null 2>&1

cd "$WORKSPACE"
claude -p "Read and execute the instructions in: $MD_FILE" \
    --append-system-prompt-file "$SYSTEM_PROMPT" \
    --dangerously-skip-permissions
```

### Step 6: Create Monitor Script

Create `$OPENCLAW_DIR/scripts/monitor.sh`:

```bash
#!/bin/bash
# Check Claude Code session status
# Auto-generated by openclaw-migration skill

BASE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
SESSION_NAME="$(basename "$BASE_DIR")"
ACTIVITY_FILE="/tmp/${SESSION_NAME}-active"

echo "=== Claude Code Monitor: $SESSION_NAME ==="
echo ""

# Check tmux session
if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
    # Get the claude process
    PANE_PID=$(tmux list-panes -t "$SESSION_NAME" -F '#{pane_pid}' 2>/dev/null | head -1)
    CLAUDE_PID=$(pgrep -P "$PANE_PID" -f "claude" 2>/dev/null | head -1)

    if [ -n "$CLAUDE_PID" ]; then
        RSS=$(ps -o rss= -p "$CLAUDE_PID" 2>/dev/null | tr -d ' ')
        RSS_MB=$((RSS / 1024))
        echo "Status:  ✅ RUNNING (PID $CLAUDE_PID, ${RSS_MB} MB)"
    else
        echo "Status:  ⚠️  tmux session exists but claude process not found"
    fi

    # Activity
    if [ -f "$ACTIVITY_FILE" ]; then
        LAST_MOD=$(stat -f %m "$ACTIVITY_FILE" 2>/dev/null || echo 0)
        NOW=$(date +%s)
        IDLE=$((NOW - LAST_MOD))
        if [ $IDLE -lt 120 ]; then
            echo "Activity: 🔴 BUSY (active ${IDLE}s ago)"
        elif [ $IDLE -lt 300 ]; then
            echo "Activity: 🟡 RECENT (${IDLE}s ago)"
        else
            echo "Activity: 🟢 IDLE ($((IDLE / 60))m ago)"
        fi
    else
        echo "Activity: — no tracking file"
    fi
else
    echo "Status:  ❌ DOWN"
fi

echo ""
echo "Attach:  tmux attach -t $SESSION_NAME"
echo "Start:   $BASE_DIR/scripts/start.sh"
```

### Step 7: Migrate OpenClaw Cron Jobs

Read the existing cron jobs from `cron/jobs.json` (if it exists):

```python
import json, os

jobs_file = "OPENCLAW_STATE_DIR/cron/jobs.json"
if os.path.exists(jobs_file):
    with open(jobs_file) as f:
        jobs = json.load(f)
    # Parse each job and suggest crontab entries
```

For each job, determine:
- **Schedule** → convert to cron format
- **Type** → does it need main session (tmux) or can run isolated?
- **Prompt/file** → what .md file or prompt to execute

If `cron/jobs.json` doesn't exist, ask:
> "Do you have any recurring tasks (heartbeat, email checks, etc.) you'd like to set up?"

Generate crontab entries and show them to the user for approval before installing.

### Step 8: Validate Skills

Scan `$WORKSPACE/skills/*/SKILL.md` and check for common issues:

```bash
for skill in "$WORKSPACE"/skills/*/SKILL.md; do
    # Check description field uses double quotes
    if grep -q '^description:' "$skill" && ! grep -q '^description: "' "$skill"; then
        echo "⚠️  $(dirname "$skill" | xargs basename): description field needs double quotes"
    fi
done
```

Fix any issues found (add quotes around description values containing colons).
